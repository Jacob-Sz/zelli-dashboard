Phase 3 · The A/B machine

Weeks 2–3 to first live test · then one experiment per week, evaluated automatically

Step 0 — instrument the funnel (without this, no test can be scored)

Six events, fired from the landing page and confirmed server-side where it counts:

EventFires whenWhy it matters
lp_viewlanding page loadsdenominator for everything
service_selecteduser picks a serviceintent signal
form_start / form_submitfirst field touched / form sentisolates the dead-click leak precisely
quote_viewprice shownthe 56% line we measured by hand
lead_capturedphone number savedthe money metric for follow-up
booked (server-side)payment confirmed in Zelliground truth — never trust the client for this

Send each as a Clarity custom event (clarity("event", ...) — free, already installed) and to the Zelli backend. ~1 dev-day. This also means the next recordings audit is automatic instead of 338 videos.

Step 1 — a tiny experiment switch, not a heavy tool

~50 lines in the landing page: hash the visitor ID → assign A or B (sticky), read the active experiment from a small config, tag every event with the variant. Client-side testing tools break exactly where your traffic lives — the Facebook/Instagram in-app browser — so we own the switch. (If you later want a UI for it, GrowthBook is the free open-source option; same idea.)

Step 2 — the automatic scorer

A script on the daily 6 AM task pulls per-variant event counts and computes conversion + significance (two-proportion test with a sequential stopping rule). It writes an Experiments card onto the dashboard — arms, conversion, probability-to-beat — and declares the result on its own:

“stop & ship B” at ≥95% probability-to-beat · “stop, no difference” after 14 days or the sample cap · otherwise “keep running”. Nobody eyeballs spreadsheets; the link you already share shows the verdict.

Honest math on speed: current volume is ~24 engaged sessions/day. Detecting a 10-point lift (56%→66% quote-reach) needs ~380 sessions per arm ≈ 4 weeks. So the process tests big swings only (one bold change, not button colors), measures the earliest metric that can move (form_start→quote_view, not booked), and the fastest lever of all: raising ad budget during a test week cuts test time proportionally. Known bugs skip the queue entirely — fixes ship, tests are for choices.
